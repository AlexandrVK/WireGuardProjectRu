# WireGuard WARP-RU AutoStart

Автоматический запуск WireGuard VPN при старте Windows с маршрутизацией только российских IP-адресов через Cloudflare WARP.

## Зачем это нужно

Cloudflare WARP — бесплатный VPN на базе WireGuard. Однако стандартная конфигурация направляет через VPN **весь** трафик, что замедляет работу с зарубежными сайтами и сервисами.

Данный проект решает эту задачу: через VPN идёт **только** трафик к российским IP-адресам (заблокированные ресурсы), всё остальное работает напрямую. При этом список российских адресов автоматически обновляется при каждом старте Windows.

## Как это работает

```
Старт Windows
     │
     ▼
Задача планировщика запускает Update-Warp-RU.ps1
     │
     ├─ Скачивает актуальный список RU IP-адресов
     ├─ Сравнивает с предыдущим (по хэшу)
     │
     ├─ Список не изменился + туннель работает  →  выход
     │
     ├─ Список не изменился + туннель не запущен  →  Install-Tunnel
     │
     ├─ Список изменился + туннель работает  →  wg syncconf (без разрыва соединения)
     │
     └─ Список изменился + туннель не запущен  →  Build-Config → Install-Tunnel
```

**Обновление маршрутов** происходит через `wg syncconf` — без остановки туннеля и без разрыва активных соединений.

## Требования

- Windows 10 / Windows 11 (x64)
- PowerShell 5.1 (встроен в Windows)
- Доступ в интернет при установке
- Права администратора

## Установка

### 1. Скачайте файлы

Скачайте архив и распакуйте в любую папку (например `C:\Downloads\warp-ru`).

В папке должны быть:
```
install.ps1
uninstall.ps1
Update-Warp-RU.ps1
sources.txt
```

### 2. Запустите установку

Правой кнопкой на `install.ps1` → **Запуск от имени администратора**

Скрипт автоматически:
- Скачает и установит WireGuard (если не установлен)
- Скачает `wgcf` для регистрации в Cloudflare WARP
- Зарегистрирует новый аккаунт WARP и создаст `warp-base.conf`
- Скопирует скрипты в `C:\WireGuardProject\`
- Создаст задачу планировщика на автозапуск при старте системы

### 3. Первый запуск VPN

После установки `install.ps1` автоматически запускает основной скрипт, который:
- Скачивает актуальный `sources.txt` с GitHub
- Скачивает список RU IP-адресов
- Устанавливает и активирует туннель warp-ru

В окне WireGuard GUI должен появиться туннель **warp-ru** в статусе Active.

Для повторного принудительного запуска:
```
C:\WireGuardProject\run-update.cmd
```

## Структура файлов после установки

```
C:\WireGuardProject\
├── Update-Warp-RU.ps1      — основной скрипт автозапуска
├── uninstall.ps1           — скрипт удаления
├── run-update.cmd          — ручной запуск для тестирования
├── warp-base.conf          — базовый конфиг WARP с ключами (не удалять!)
├── warp-ru.conf            — генерируемый конфиг с RU маршрутами
├── warp-peer.conf          — временный файл для wg syncconf
├── sources.txt             — список URL источников RU IP (синхронизируется с GitHub)
├── ru-last.txt             — последний скачанный список RU IP
├── ru-last-run.txt         — дата последнего успешного запуска
├── ru-new.txt              — временный файл при скачивании
└── warp.log                — лог работы (последние 200 строк)
```

> **Важно:** `warp-base.conf` содержит ваши приватные ключи WARP. Не публикуйте этот файл.

## Источники списка RU IP-адресов

Список источников хранится в отдельном файле `sources.txt` в репозитории:
[AlexandrVK/WireGuardProjectRu/sources.txt](https://github.com/AlexandrVK/WireGuardProjectRu/blob/main/sources.txt)

При каждом запуске скрипт **автоматически синхронизирует** локальный `sources.txt` с GitHub.
Если нужно добавить или заменить источник — достаточно отредактировать файл на GitHub,
основной скрипт трогать не нужно.

Формат файла: один URL на строку, строки с `#` — комментарии. Скрипт пробует источники
по порядку, использует первый успешный.

Текущие источники:
1. `ipverse/country-ip-blocks` — данные из всех 5 RIR, агрегированные, обновляются ежедневно
2. `Travisun/Latest-Country-IP-List` — обновляется ежедневно
3. `ebrasha/cidr-ip-ranges-by-country` — резервный источник

## Как устроен туннель

WireGuard в Windows управляет туннелями через **WireGuard Manager** — фоновую службу. При установке туннеля:

1. Конфиг `.conf` копируется в `C:\Program Files\WireGuard\Data\Configurations\`
2. Manager шифрует его в `.conf.dpapi` и удаляет оригинал
3. Запускается именованная служба `WireGuardTunnel$warp-ru`

Статус туннеля проверяется через `wg show warp-ru` — прямой запрос к WireGuard без косвенных проверок через SCM.

## Обновление маршрутов на лету

Если туннель уже работает и список RU IP изменился, скрипт применяет новые маршруты через:

```
wg syncconf warp-ru warp-peer.conf
```

`wg syncconf` обновляет только изменившиеся параметры и **не разрывает** текущие соединения. Туннель продолжает работать без перерыва.

## Ручное тестирование

### Принудительный запуск основного скрипта
```
C:\WireGuardProject\run-update.cmd
```
Запускает тот же 64-битный PowerShell что и планировщик задач.

### Тест wg syncconf
```powershell
# Правый клик → Запуск от имени администратора
C:\WireGuardProject\test-syncconf.ps1
```
Заменяет AllowedIPs на базовый список из `warp-base.conf` и выводит `wg show` до и после.

Для восстановления полного RU списка после теста — удалите `ru-last.txt` и запустите `run-update.cmd`:
```powershell
Remove-Item C:\WireGuardProject\ru-last.txt -Force
```
Без удаления `ru-last.txt` скрипт решит что список не изменился и ничего не применит.

### Проверка статуса туннеля
```powershell
C:\WireGuardProject\test-tunnel-status.ps1
```

### Проверка маршрутизации

Поскольку используется split tunnel (через VPN идут **только** RU-адреса), проверять нужно иначе — убедиться что трафик к конкретному RU-адресу действительно уходит через интерфейс `warp-ru`.

**Способ 1 — через route print:**
```powershell
# Смотрим таблицу маршрутов — должны быть маршруты через интерфейс warp-ru
route print | Select-String "warp"
```

**Способ 2 — Find-NetRoute для конкретного RU IP:**
```powershell
# Берём любой IP из RU списка и проверяем через какой интерфейс пойдёт пакет
# Например 5.8.8.0 — первая подсеть из типичного RU списка
Find-NetRoute -RemoteIPAddress "5.8.8.8" | Select-Object InterfaceAlias, NextHop
```
Если маршрутизация работает — `InterfaceAlias` покажет `warp-ru`, а не ваш основной сетевой адаптер.

**Способ 3 — wg show:**
```powershell
& "C:\Program Files\WireGuard\wg.exe" show warp-ru
```
В поле `transfer` должны расти счётчики rx/tx по мере использования сети.

## Удаление

Правый клик на `C:\WireGuardProject\uninstall.ps1` → **Запуск от имени администратора**

Скрипт удалит:
- Туннель warp-ru (если активен)
- Задачу планировщика `WireGuard-WARP-RU`
- WireGuard (программа, через MSI)
- Папку `C:\WireGuardProject` со всеми файлами

## Устранение проблем

### Туннель не запускается при старте

Проверьте лог: `C:\WireGuardProject\warp.log`

Убедитесь что задача планировщика существует:
```powershell
Get-ScheduledTask -TaskName "WireGuard-WARP-RU"
```

### WireGuard показывает ошибку в GUI

Проверьте что `warp-base.conf` содержит валидные ключи WARP. Если аккаунт был деактивирован Cloudflare — удалите `warp-base.conf` и запустите `install.ps1` заново для регистрации нового аккаунта.

### Скрипт видит неправильный путь к WireGuard

Убедитесь что задача планировщика пересоздана после последнего `install.ps1`. Старые задачи могли использовать 32-битный PowerShell. Запустите `install.ps1` ещё раз — задача будет пересоздана с правильными параметрами.

### Нет доступа к российским сайтам через VPN

Проверьте через `wg show warp-ru` что в поле `allowed ips` присутствуют российские подсети. Запустите `run-update.cmd` для принудительного обновления списка.
